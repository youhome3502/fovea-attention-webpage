Project Overview

Research Motivation

The human visual system processes visual information with remarkable efficiency. The fovea - a small pit in the retina about 5.5mm in diameter - provides our high-acuity central vision, while peripheral vision provides broader context at lower resolution. This biological architecture has evolved over millions of years to optimize visual processing under computational constraints.

This research investigates whether similar computational principles can improve artificial neural networks for visual recognition tasks. Specifically, we explore:

  • Efficient attention: Can processing only part of the image at high resolution reduce computation while maintaining accuracy?
  • Adaptive processing: Should the region of interest be fixed at center, or learned dynamically?
  • Uncertainty awareness: Can models知道自己 "know what they don't know"?
  • Multi-scale fusion: How should different resolution levels be combined?

Key Hypothesis

Fovea-inspired attention mechanisms can significantly improve deep learning models by selectively processing informative regions at high resolution while maintaining contextual awareness through peripheral processing.

Module 1: Plain ResNet18 Baseline 60.00%

Research Motivation

Every meaningful experiment requires a control group. Before we can claim that fovea attention provides any benefit, we need a reliable baseline that measures what vanilla architectures achieve on CIFAR-10. Plain ResNet18, without any attention mechanisms, serves as this essential reference point. This module establishes the performance ceiling that our fovea-inspired approaches must exceed to demonstrate value.

The baseline is particularly important because it reveals the inherent difficulty of CIFAR-10 classification. With only 32x32 pixel images and 10 balanced classes, 60% accuracy might seem low compared to ImageNet results, but this reflects the limited visual information available in low-resolution images.

Technical Principle

ResNet18 consists of 4 residual blocks (layer1-4) with skip connections. Each block contains two convolutional layers with batch normalization and ReLU activation. The architecture progressively increases channels (64→128→256→512) while reducing spatial dimensions. Average pooling followed by a linear layer produces final classifications.

Input(32x32) → Conv1+BN+ReLU → Layer1(64) → Layer2(128) → Layer3(256) → Layer4(512) → AvgPool → FC(10)

Implementation Details

from torchvision.models import resnet18
model = resnet18(num_classes=10)
# Standard training: Adam(lr=0.001), batch_size=128, epochs=20

Experimental Results

With 20 epochs of training, plain ResNet18 achieves 82.20% test accuracy. Initial experiments with 5 epochs showed 60%, demonstrating that extended training significantly improves results. This finding is crucial: longer training is essential for fair comparison between architectures.

Comparison with Other Modules

Plain ResNet18 treats all image regions equally. It lacks any mechanism for selective attention, meaning computational resources are wasted on background pixels. Our fovea modules aim to improve upon this by focusing processing on semantically important regions.

Module 2: FoveaResNet (Fixed Center) 68.99%

Research Motivation

Humans and primates evolved the fovea because not all visual information is equally important. The center of gaze typically contains the object of interest, while peripheral vision provides context. This module tests the fundamental hypothesis: can mimicking this architecture improve neural networks?

The choice of a fixed center (always processing the image center at high resolution) is deliberate. Before exploring complex learned attention mechanisms, we must establish whether the basic principle of hierarchical resolution provides benefits. If fixed center already improves results, learned attention has a strong foundation to build upon.

Technical Principle

FoveaResNet processes the image in two streams: (1) Center path: Extracts high-resolution features from an 8x8 central patch. (2) Periphery path: Downsamples the entire image 2x, extracts features, then upsamples back. Features are combined via element-wise addition.

Input → Conv1+L1+L2 → [Center: 8x8 full-res | Periphery: 2x down → process → upsample] → Add → L3+L4 → Output

Implementation Details

def fovea_attention(x, center_size=8):
    # Periphery: downsample, process, upsample
    periph = F.avg_pool2d(x, kernel_size=2)
    periph_feat = periph_conv(periph)
    periph_up = F.interpolate(periph_feat, size=(H,W))
    
    # Center: extract and process 8x8 patch
    center_patch = x[..., h-half:h+half, w-half:w+half]
    center_feat = center_conv(center_patch)
    center_full = F.interpolate(center_feat, size=(H,W))
    
    # Combine with spatial mask
    output = mask * center_full + (1-mask) * periph_up
    return output

Critical bug fix: Initial implementation used torch.zeros_like(x) for padding, which destroyed peripheral features. The fix uses spatial masking to preserve periphery while adding high-resolution center features.

Module 9: Learnable Fovea with GlimpseLocator 63.63%

Research Motivation

If fixed center fovea improves performance, perhaps optimal positioning could improve further. Not all interesting objects are centered - sometimes the subject is off-center. This module explores whether the network can learn to predict where to look based on image content.

The GlimpseLocator network is inspired by saccadic eye movements in biological systems. Before making a decision about what an image contains, the visual system rapidly shifts gaze to informative regions. Can artificial systems learn similar behavior?

Technical Principle

GlimpseLocator processes feature maps through spatial attention to produce a probability distribution over image locations. Soft-argmax computes a differentiable "center of attention" that can be trained via backpropagation. This location then guides fovea positioning.

Input → ResNet Backbone → [GlimpseLocator: Spatial Attention + Soft-Argmax] → Location(cx,cy) → Fovea(learned center)

Implementation Details

class GlimpseLocator(nn.Module):
    def forward(self, features):
        # Spatial attention over locations
        attn = self.spatial_attention(features)  # [B, 1, H, W]
        
        # Soft-argmax for differentiable center
        attn_flat = F.softmax(attn.view(B, -1), dim=1)
        grid_y, grid_x = torch.meshgrid(...)
        
        cx = (attn_flat * grid_x).sum()
        cy = (attn_flat * grid_y).sum()
        
        return torch.stack([cx, cy], dim=1)  # [B, 2]

The location prediction is trained with a center-biased self-supervised loss, encouraging predictions toward image center while allowing deviation when evidence supports it.

Experimental Results

Surprisingly, learnable fovea (63.63%) < fixed center (68.99%). Analysis revealed that GlimpseLocator quickly converges to predicting near-center locations, suggesting that for CIFAR-10 images, the optimal gaze position is typically close to center anyway.

Key Finding: For CIFAR-10 with centered objects, fixed center is already near-optimal. Learned positioning provides greater benefits when objects are frequently off-center.

Comparison with Other Modules

Despite lower accuracy, learnable fovea provides valuable insights: (1) Network can learn spatial attention, (2) CIFAR-10 objects are typically centered, (3) Fixed center is a strong baseline. This module informs future work on when learned attention is beneficial.

Module 3: UncertaintyViT with MC Dropout 72.40%

Research Motivation

Beyond attention mechanisms, we wondered: can models quantify their uncertainty? In critical applications (medical diagnosis, autonomous vehicles), knowing when the model is uncertain is as important as getting the prediction right. This module combines Vision Transformers with Bayesian uncertainty estimation.

The choice of ViT architecture reflects its success in modern computer vision. Transformers process images as sequences of patches, naturally supporting attention mechanisms. MC Dropout provides a practical way to estimate uncertainty without expensive ensemble training.

Technical Principle

Monte Carlo Dropout runs multiple forward passes with dropout enabled. The mean across passes serves as prediction, while standard deviation indicates uncertainty. This approximates Bayesian inference, treating network weights as distributions rather than point estimates.

Input → Patch Embed → Transformer Blocks (with dropout) → [x5 MC passes] → Mean=Prediction, Std=Uncertainty

Implementation Details

def mc_forward(self, x, n_passes=5):
    self.train()  # Enable dropout
    logits_list = []
    for _ in range(n_passes):
        with torch.no_grad():
            logits_list.append(self(x))
    
    logits_stack = torch.stack(logits_list)
    mean = logits_stack.mean(dim=0)
    uncertainty = logits_stack.std(dim=0)
    
    return mean, uncertainty

5 passes provide good uncertainty estimates while remaining computationally tractable. The uncertainty head predicts log-variance, providing calibrated uncertainty estimates.

Experimental Results

UncertaintyViT achieves 72.40% test accuracy, representing a +12.4% improvement over baseline. Notably, uncertainty values increase during training (from 0.16 to 0.38), suggesting the model becomes more calibrated as training progresses.

Key Finding: ViT architecture significantly outperforms ResNet for this task, and uncertainty estimation provides confidence scores without accuracy penalty.

Comparison with Other Modules

UncertaintyViT (72.40%) surpasses FoveaResNet (68.99%) in accuracy. This suggests that (1) ViT architecture is better suited for CIFAR-10, and (2) uncertainty estimation may provide regularization benefits. However, it lacks explicit spatial attention.

Module 4: MRAM (Multi-Resolution Adaptive Mapping) 66.06%

Research Motivation

What if we process multiple resolution levels simultaneously? Instead of just center vs periphery, MRAM uses a 3-level pyramid (1x, 2x, 4x downsampling) with learnable fusion weights. This explores whether finer granularity in resolution levels improves feature extraction.

The pyramid architecture is inspired by image processing techniques and biological visual systems that process information at multiple scales. By learning optimal fusion weights, the network can adaptively emphasize the most useful resolution levels for the task.

Technical Principle

Three parallel streams process the input at different resolutions. Each stream applies convolutional processing, then all streams are upsampled to original size. Learnable softmax-normalized weights determine fusion: w1=0.61, w2=0.23, w3=0.17.

Input → [Level1(full-res) → conv] + [Level2(2x↓) → conv → up] + [Level3(4x↓) → conv → up] → [Softmax(w1,w2,w3)] → Fusion → Transformer → Output

Implementation Details

class MultiResolutionPyramid(nn.Module):
    def forward(self, x):
        level1 = self.conv1(x)  # Full resolution
        level2 = F.avg_pool2d(x, 2)
        level2 = self.conv2(level2)
        level2 = F.interpolate(level2, size=x.shape[2:])
        
        # Learnable weighted fusion
        weights = F.softmax(self.level_weights, dim=0)
        output = weights[0]*level1 + weights[1]*level2 + weights[2]*level3
        return output, weights

Experimental Results

MRAM achieves 66.06% test accuracy. The learned weights show clear preference: w1=61% (highest resolution) > w2=23% > w3=17%. This confirms that higher resolution features are most informative for CIFAR-10 classification.

Observation: While multi-resolution processing shows promise, performance is lower than simpler two-level (center/periphery) approaches. The additional complexity may not justify the benefit for small images.

Comparison with Other Modules

MRAM (66.06%) underperforms FoveaResNet (68.99%) and UncertaintyViT (72.40%). This suggests that for 32x32 images, simpler architectures suffice. Multi-resolution processing may provide greater benefits on larger images (ImageNet) where scale variation is more significant.

Module 13: Hybrid (MRAM + Uncertainty) 57.24%

Research Motivation

If MRAM provides multi-resolution processing and UncertaintyViT provides uncertainty estimation, perhaps combining both could yield synergistic benefits? This module tests whether the advantages of different architectures can be combined through integration.

However, combining complex mechanisms requires careful design. Adding multiple components without proper initialization or training strategy can lead to interference, where components "fight" rather than cooperate.

Technical Principle

The hybrid combines MRAM's multi-resolution pyramid with UncertaintyViT's classification and uncertainty heads. The architecture takes MRAM's fused features and routes them through the uncertainty estimation pipeline.

Input → 3-Level Pyramid (MRAM) → Transformer → [Classification Head + Uncertainty Head]

Experimental Results

Surprisingly, the hybrid underperforms both parent architectures! 57.24% < MRAM (66.06%) and 57.24% < UncertaintyViT (72.40%). This demonstrates a negative interaction effect.

Critical Lesson: Simply combining complex mechanisms does not guarantee improvement. Multi-task learning with incompatible objectives can degrade performance. Proper architecture design and training strategies are essential.

Analysis

Several factors likely contribute to poor performance: (1) Training instability from conflicting objectives, (2) Insufficient capacity for both tasks, (3) Lack of proper initialization strategy, (4) No curriculum learning to gradually introduce complexity.

Comparison with Other Modules

This negative result is informative: it shows that architectural "additions" must be carefully designed. Future work should explore gradual integration or shared representations between components.

Module 14: RL-Based Gaze Control (BEST) 84.80%

Research Motivation

The ultimate goal of fovea attention is active, intelligent gaze selection. Rather than relying on fixed positions or simple heuristics, can the system learn to look at the most informative image regions? This module applies reinforcement learning (PPO) to optimize fovea positioning end-to-end.

Reinforcement learning provides a natural framework for gaze control: (1) The policy network selects gaze positions, (2) The environment (backbone + classifier) provides rewards based on classification accuracy, (3) Policy gradient methods optimize gaze selection directly.

Technical Principle

Proximal Policy Optimization (PPO) with clipped surrogate objective enables stable policy learning. The policy network outputs continuous gaze coordinates (cx, cy). The reward is classification accuracy, with exploration encouraged through entropy bonus.

Image → FoveaResNet(backbone,frozen) → Features → [PPO Policy → Glimpse] → Classification → Reward → PPO Update

Implementation Details

class GlimpsePolicy(nn.Module):
    def forward(self, features):
        # Encode features
        x = self.encoder(features)
        
        # Output gaze coordinates
        mean = torch.tanh(self.mean_head(x))  # [-1, 1]
        std = torch.exp(self.log_std)
